In simple terms, classes and IDs are used in HTML and CSS to identify and style elements on a web page. Here are some best practices associated with using classes and IDs:

  1. Classes
    • Classes are used to group elements that share similar characteristics or styles. For example, you can have a class called "button" and apply it to multiple buttons on your page.
    • Best practice is to use classes when you have multiple elements that need to be styled in the same way. It allows you to apply styles to multiple elements at once without repeating code.
    • Classes are not unique, meaning you can use the same class name on different elements throughout your page or even across different pages.

    STEP by STEP guide:

    1. Create a CSS class attribute declaration. Note to start with "." <style> .class-name { color: blue; } </style>
    2. It is now time to call & apply the class name create on step 1. Apply the class to an element like this: <h1 class="class-name"> Hello World</h2>

  2. ID
    • IDs are used to uniquely identify a specific element on a web page. Each ID should only be assigned to one element.
    • Best practice is to use IDs when you need to target a specific element for a particular purpose. For example, you might have an element that requires specific JavaScript functionality or unique styling.
    • IDs are intended to be unique within the entire document, so it's not recommended to reuse the same ID on multiple elements.

    STEP by STEP guide:

    1. Create a CSS id attribute declaration. Note to start with "#". <style> .id-name { color: blue; } </style>
    2. It is now time to call & apply the Id name create on step 1. Apply the Id to an element like this: <h1 id="id-name"> Hello World</h2>
